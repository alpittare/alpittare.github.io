Phase 7: Validation & Testing Checklist

This section outlines critical validation checks that should be performed during platform consolidation to ensure consistency, performance, and correctness across all games.

Physics & Core Engine Validation

  • Fixed Timestep Verification — All games (except Voyager/Baseball/Survival) should use 60Hz fixed timestep. Test deterministic replay: record 100 frames, replay, verify frame-by-frame match.
  • Delta Time Clamping — Verify max delta is enforced (0.016-0.1s range). Simulate 500ms freeze frame; confirm game continues without physics explosion.
  • Entity Lifecycle — Test deferred add/remove: spawn 1000 entities across 60 frames, verify no crashes, confirm update/render order correct.
  • Collision Detection Accuracy — Run AABB stress test: 500 bodies, 10K collisions/frame. Measure latency, verify no false negatives.
  • Interpolation Smoothness — Enable interpolated rendering; record 1-second video at 60fps. Verify no jitter, smooth curves through world boundary crossings.

UI & Rendering Validation

  • Consistent Glass-Card Style — All games use same card component. Test across 3 screen sizes; verify spacing, border-radius, opacity match exactly.
  • Neon Rect Rendering — Verify #00ff88 / #00ccff colors render identically. Screenshot regression on 5 devices; pixel-perfect match required.
  • Font Caching — Measure render time with 100 text elements. Should be <2ms. Verify font cache hits 95%+ on second frame.
  • Gradient Performance — Test 50 concurrent gradients (status bars, buttons). Should run at 60fps on 2022 mid-range phones.
  • Accessibility Contrast — Run WCAG contrast checker on all UI. Verify WCAG AA (4.5:1 for text) or AAA (7:1) for critical elements.

Save System Validation

  • Version Migration Path — Test v1→v2→v3 migrations. Simulate data loss scenarios (corrupt JSON, missing fields). Verify graceful fallback to default state.
  • Quota Handling — Fill localStorage to 90% capacity. Verify save system detects quota exceeded, alerts user, prioritizes critical data.
  • Save Throttling — Spam save calls 1000×/sec. Verify throttle buffer (300ms) prevents browser lag; no duplicate writes.
  • Capacitor Persistence — On iOS/Android, force background kill. Verify Capacitor Preferences survives app restart (compare vs localStorage).
  • Data Integrity — Corrupt save file manually. Verify load function detects corruption, restores from backup, logs error.

Performance Validation

  • Particle Pool Caps — All games limit particle pools (500-1000). Test overflow: spawn 2000 particles. Verify only N active, oldest culled gracefully.
  • Memory Profiling — Run each game for 5 minutes. Verify heap growth <50MB, no GC stutters >16ms, PSS stable after 30 seconds.
  • Frame Rate Consistency — Record 1-minute gameplay session. Verify 95th percentile frame time <16.7ms (60fps), <50 jank frames.
  • Build Size Optimization — Measure minified+gzipped size for shared core. Should be <100KB. Per-game module <50KB each.

AI System Validation

  • PlayerModel Accuracy — Play 100 games vs AI. Verify AI correctly predicts player behavior with >70% accuracy by game 50.
  • Adaptive Difficulty Convergence — Set target win rate (50%). Play 100 games. Verify win rate converges to target within ±5%.
  • Bayes Predictor Calibration — Test on Cricket & Football. Verify predicted probabilities match observed frequencies (Brier score <0.2).
  • Claude API Rate Limiting — Verify advisor calls cached, max 1 call/5 minutes. Stress test with 100 concurrent games; no quota exceeded errors.

Input & Interaction Validation

  • Screen→World Transform — Test on 3 aspect ratios (18.5:9, 16:9, 20:9). Verify tap position maps to correct world coordinate ±2 pixels.
  • Touch Swipe Classification — Test 50 swipes (up/down/left/right). Verify >95% accuracy. Edge cases: diagonal, slow, fast.
  • Gesture Recognition — Test pinch zoom, two-finger drag. Verify sensitivity consistent across devices, no phantom gestures from single touch.
  • DPI Scaling — Test on 1x, 2x, 3x DPI devices. Verify HUD elements size correctly (44pt buttons for touch targets).

Accessibility Validation

  • Color Blind Modes — Test protanopia, deuteranopia, tritanopia. Verify no game-critical info conveyed by color alone. Use Coblis simulator.
  • Reduced Motion — Enable prefers-reduced-motion. Verify animations reduced by 50%, opacity/scale preferred over translate.
  • Large Touch Targets — Verify all interactive elements ≥44×44pt. Test one-handed use on 6.1" phone; no reaches beyond thumb zone.
  • High Contrast — Enable high contrast mode. Verify all text ≥4.5:1 ratio, borders visible around all buttons.
  • Screen Reader — Test with VoiceOver (iOS) / TalkBack (Android). Verify all interactive elements labeled, read in logical order.

Deployment Validation

  • PWA Installation — Test on Chrome, Samsung Internet, Edge. Verify manifest valid, service worker caches correctly, offline mode works.
  • Capacitor Build — Build for iOS/Android. Verify no native errors, app launches, native features (haptics, audio context) respond.
  • CI/CD Pipeline — Run GitHub Actions workflow. Verify all 6 games build without errors, unit tests pass, artifacts upload.
  • IAP Integration — Test purchase flow on iOS/Android testflight. Verify receipt validation, coins awarded, no duplicate purchases.

Multiplayer Validation

  • Room Creation/Join — Create 100 rooms, join/leave 1000 times. Verify no stale rooms, join times <500ms, list accuracy.
  • State Synchronization — Play 10 matches; capture network traffic. Verify position updates arrive <100ms latency, no jitter >50px.
  • Spectator Mode — Watch 10 live games. Verify spectator sees live position updates, chat works, disconnect handling graceful.

Phase 8: Scaling Recommendations

As your game portfolio grows, these architectural and operational recommendations will help maintain code quality, development velocity, and system reliability at scale.

Build & Deployment

  1. Module Bundler Strategy — Adopt Vite or esbuild for composition. Enable tree-shaking to reduce bundle size. Create entry points:
    // package.json exports "exports": { "./core": "./dist/core/index.js", "./systems": "./dist/systems/index.js", "./cricket": "./dist/games/cricket/index.js", "./football": "./dist/games/football/index.js" }
  2. Game Module Size Target — Each game should be 1-2K lines (game-specific logic only). Target final bundle: 10-15KB per game (core shared 100KB).
  3. Hot Reload Development — Implement Vite dev server with HMR. Shared core changes propagate instantly to all games in dev mode. Game module hot-reload. Velocity boost
  4. Unified CI/CD Pipeline — One GitHub Actions workflow builds all 5 games + core tests:
    • Lint (ESLint)
    • Test (Jest for core, headless Chrome for games)
    • Build (Vite production build)
    • Upload artifacts (S3)
    • Deploy to staging (Vercel)
    Save 5+ hours/release

Testing & Quality Assurance

  1. Automated Physics Simulation Testing — Create agent that plays each game 1000 times. Verify:
    • No crashes in 1000 games
    • Physics determinism (replay matches)
    • AI difficulty within target range
    • Save/load integrity
  2. UI Screenshot Regression Testing — Capture screenshots of all screens on 3 device sizes (375×667, 414×896, 540×720). Compare against baseline with image diffing. Catch layout regressions automatically.
  3. Performance Profiling Automation — Run Lighthouse/WebPageTest on each game. Track metrics:
    • FCP (First Contentful Paint) target: <1s
    • TTI (Time to Interactive) target: <2s
    • Avg frame time: <16.7ms
    • Memory PSS: <100MB
  4. Load Testing Multiplayer — Simulate 100 concurrent players in room system. Verify:
    • State sync latency <100ms (p95)
    • No memory leaks over 1 hour
    • Graceful degradation above capacity

Backend & Analytics

  1. Unified Backend (Convex or Firebase) — Consolidate leaderboards, multiplayer, analytics into single backend:
    • Leaderboards: One table, game-scoped queries
    • Multiplayer: Single room system, game-specific state
    • Analytics: Unified event schema across all games
    • Auth: One OAuth/Google Play login per user
  2. Analytics Event Standardization — Define common event schema:
    { "event": "level_complete", "game": "cricket", "userId": "u123", "timestamp": 1609459200, "data": { "levelId": 42, "levelTime": 180, "difficulty": "hard", "score": 5000 } }
    Track across all games for cohort analysis, retention metrics, monetization dashboards.
  3. Real-Time Monitoring Dashboard — Grafana dashboard tracking:
    • Crash rate per game
    • Active players (per game, cohort)
    • Revenue (IAP, ads, battle pass)
    • Performance (FCP, TTI, frame time)
    • Multiplayer queue depth

Team & Process

  1. Module Ownership Model — Assign owners to core modules:
    • Physics: Owner must sign off on changes
    • AI Engine: Owner must test adaptive difficulty
    • Save System: Owner must verify migration paths
    • Audio: Owner must test iOS context resume
    Prevents cascading failures from careless refactors.
  2. Code Review Standards — Require peer review for:
    • Any core/ changes (physics, engine, input)
    • Any save format changes (migration path required)
    • Any performance-critical paths (profiler output required)
  3. Documentation-as-Code — Each module has inline JSDoc + Architecture.md describing:
    • Public API
    • Constraints (e.g., "must be called after engine.init()")
    • Performance characteristics
    • Known limitations
  4. Game-Specific Customization Patterns — Document how games customize platform modules:
    • Physics: Override gravity, collision layers
    • Progression: Extend with game-specific achievements
    • AI: Plug in custom bot personalities
    • Audio: Register custom sound effects

Future Expansion (Year 2+)

  1. 6th Game (New Game Launch) — Should take 4-6 weeks (80% time saved vs building from scratch). Use platform template: inherit 90% of systems, implement 500 lines of game-specific logic.
  2. Cross-Game Progression — Unified XP system allowing players to earn coins in Cricket, spend in Football. Single battle pass for all games. Requires backend coordination.
  3. Platform SDK — Package platform as npm module for external developers. Enable third-party game studios to build games on your platform. Revenue-sharing model.
  4. AI Advisor Assistant — Scale Claude API integration beyond Cricket. Per-game coaching: "Try medium-pace bowl" in Cricket, "Aim for far corner" in Football.
  5. Social Features — Implement friend leaderboards, team battles, tournament brackets. Requires multiplayer backend scaling.
