The AI Systems Behind the Threat

Our enemies aren't scripted. Four core systems work together to create truly intelligent adversaries:

🐦 Swarm Intelligence

Boid-like flocking with tactical overlays for coordinated movement. Enemies move like a flock of birds, maintaining formation while responding to tactical goals.

🌳 Behavior Trees

Each enemy type has a behavior tree defining priorities and decision-making. Behavior trees allow enemies to switch between attacking, retreating, patrolling, and flanking based on dynamic conditions.

📍 Tactical Coordinator

A central AI system assigns roles (attacker, flanker, distractor) to groups of enemies. This ensures coordinated strategies rather than individual behaviors.

🎓 Player Adaptation

Tracks player's preferred position, weapon choice, and movement patterns. Over time, enemies learn your habits and adjust their strategies accordingly.

Technical Deep Dive

Behavior Trees: Decision Making from the Ground Up

Each enemy archetype uses a behavior tree—a hierarchical decision-making structure that evaluates conditions and selects actions in priority order.

Basic Structure:

Root → Selector [Attack, Retreat, Patrol, Flank] → each with conditions and actions

Here's a simplified example of a rusher enemy's behavior tree node in pseudocode:

class BehaviorNode {
    execute(enemy, context) {
        if (context.playerDistance < ENGAGE_RANGE) {
            return this.attack(enemy, context);
        } else if (context.health < RETREAT_THRESHOLD) {
            return this.retreat(enemy, context);
        } else if (context.canFlank) {
            return this.flank(enemy, context);
        } else {
            return this.patrol(enemy, context);
        }
    }

    attack(enemy, context) {
        // Move toward player, dodge obstacles
        enemy.position += moveToward(context.player, enemy.speed);
        enemy.fire(context.player);
        return Status.RUNNING;
    }

    flank(enemy, context) {
        // Calculate flank position
        let flankPos = calculateFlankPosition(
            context.player,
            enemy.position,
            FLANK_DISTANCE
        );
        enemy.position += moveToward(flankPos, enemy.speed);
        return Status.RUNNING;
    }
}

A* Pathfinding with Tactical Layers

Standard A* pathfinding augmented with influence maps: a 2D grid scoring system where each cell is evaluated for tactical advantage.

Grid Scoring:

Each cell accumulates scores from: safety (distance from player's weapon range), ally positions (bonus for proximity), cover positions (bonus for defensive spots).

Enemies don't just find the shortest path—they find the path that maximizes tactical advantage.

Influence Maps: Spatial Strategy

The tactical backbone of our AI. Influence maps are 2D grids where each cell represents tactical value:

Influence Map Concept
Player Position: X (weapon range = 3 cells)

    [  ] [-1] [-2] [-3] [-2] [-1] [  ]
    [-1] [-2] [-3] [-4] [-3] [-2] [-1]
    [-2] [-3] [-4] [ X] [-4] [-3] [-2]
    [-1] [-2] [-3] [-4] [-3] [-2] [-1]
    [  ] [-1] [-2] [-3] [-2] [-1] [  ]

Legend:
  X  = Player (danger zone)
  -N = Danger level (negative = bad, enemy avoids)
  [  ] = Neutral (safe, but no tactical advantage)

Enemies calculate path to LOWEST value (safest)
while maintaining line of sight or cover.

Example: Enemy chooses flanking route (value +2)
over direct approach (value -3) for tactical advantage.

Wave Difficulty Scaling

Difficulty scales exponentially based on wave number and player performance:

Formula:

waveStrength = baseStrength × (1 + log(wave) × difficultyMultiplier) + adaptationBonus

Where adaptationBonus increases when the player dominates a wave, ensuring progressive challenge without sudden difficulty spikes.

Flocking + Tactical Movement

Enemies use Reynolds boids (separation, alignment, cohesion) combined with tactical goals:

  • Separation: Enemies maintain spacing to avoid collision
  • Alignment: Move in coordinated directions as a group
  • Cohesion: Attracted to the center of the swarm
  • Tactical Goals: Overlay flocking with tactical priorities (attack, flank, distract)

This creates realistic group movement that appears intelligent and coordinated, not scripted.
