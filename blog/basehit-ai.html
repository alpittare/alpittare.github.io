Technical Deep Dive

For the technically curious, here's how these systems work in detail:

Markov Chain Pitch Selection

The core of the pitch sequencing engine uses a Markov chain model. Each state represents the current situation:

State = (pitch_count, last_pitch_type, batter_tendency) → Next Pitch Probability

Example:
  (0-1, Fastball, Pull_Hitter) → {
    Curveball: 0.45,      // Setup pitch, attack weakness
    Fastball: 0.30,       // Get ahead
    Slider: 0.25          // Putaway pitch
  }

  (0-2, Curveball, Pull_Hitter) → {
    Fastball_High: 0.65,  // Putaway - out of zone
    Slider: 0.35          // Alternative putaway
  }

Transition probabilities are optimized to minimize expected run value—the pitcher always chooses the pitch most likely to prevent runs.

Expected Run Value

Every pitch decision is evaluated against one metric: expected run value. This is the predicted number of runs the opposing team will score in the current inning if a given pitch is thrown.

  • High expected run value: Risky pitch that could lead to runs
  • Low expected run value: Safe pitch that limits run potential
  • Decision rule: Always choose the pitch that minimizes expected run value

Difficulty as Pitcher Skill

Difficulty doesn't make the AI "smarter"—it makes the AI more precise:

  • Easy: Large variance around intended pitch location; high miss rate
  • Normal: Moderate variance; realistic control
  • Hard: Tight variance; pinpoint command that's nearly impossible to exploit
  • Legendary: Ultra-tight variance; perfect execution within fractions of an inch

This creates a natural difficulty curve: a hard pitcher is harder not because they're strategic geniuses, but because they execute their strategy with near-perfect precision.

Key Insight: The pitch selection strategy is identical across all difficulties. What changes is the variance in execution. This ensures that learning to read a pitcher's strategy on Easy carries over to Hard—you're just playing against a more precise version of the same intelligence.

Pitch Selection Algorithm (Simplified)

function selectPitch(gameState) {
  // Step 1: Get batter profile
  const batterProfile = getBatterProfile(gameState.batter);

  // Step 2: Look up transition probabilities
  const state = {
    count: gameState.count,
    lastPitch: gameState.lastPitch,
    tendency: batterProfile.tendency
  };

  const pitchProbabilities =
    markovChain[state.toString()];

  // Step 3: Evaluate expected run value
  let bestPitch = null;
  let lowestERV = Infinity;

  for (const pitch of Object.keys(pitchProbabilities)) {
    const erv = calculateExpectedRunValue(
      pitch,
      batterProfile,
      gameState
    );

    if (erv < lowestERV) {
      lowestERV = erv;
      bestPitch = pitch;
    }
  }

  // Step 4: Add execution variance based on difficulty
  const variance = getDifficultyVariance(
    gameState.difficulty
  );
  const location = addNoise(
    pitchLocations[bestPitch],
    variance
  );

  return {
    type: bestPitch,
    location: location,
    velocity: addNoise(pitchVelocities[bestPitch], variance)
  };
}
